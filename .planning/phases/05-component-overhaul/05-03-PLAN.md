**New layout structure:**

1. **Section header** -- Left-aligned (not centered):
   - Title in `text-4xl md:text-5xl font-heading font-bold text-foreground` -- left-aligned
   - Accent word in `text-[var(--color-accent)]`
   - Subtitle below in `text-lg` with `style={{ color: 'var(--text-secondary)' }}`
   - "View All Case Studies" link below subtitle, left-aligned, with `font-mono text-sm`
   - Use `mb-16` gap between header and list

2. **Project list** -- Vertical stack of expandable cards:
   - Each project is a full-width row/card
   - **Collapsed state** (default for all except first project): Shows a horizontal row with:
     - Left: Project title in `text-xl md:text-2xl font-heading font-bold`
     - Center: First 3 tech stack items as `font-mono text-sm` badges
     - Right: Role label in `font-mono text-sm` with `text-[var(--text-secondary)]` + expand chevron
     - The row is clickable to expand
     - Bottom border: `border-b border-[var(--color-border)]`

     - Same header row but with collapse chevron (rotated)
     - Below the header row, show the detail content in a 2-column asymmetric grid: `grid grid-cols-1 lg:grid-cols-5 gap-8`
     - Left column (3/5 = `lg:col-span-3`): Business problem, role/responsibilities, technical solution with key decisions

     - Detail area padding: `pt-6 pb-8`

3. **State management:**
   - Use `useState<string | null>` to track which project is expanded (store project id, null = none)
   - Initialize with first project id expanded: `useState<string | null>(projects[0].id)`

   - Clicking a collapsed project expands it (set its id)

4. **Detail content layout (expanded):**

   - Technical solution: architecture label in `font-mono text-sm`, tech stack as `font-mono` badges in `bg-[var(--color-surface-sunken)] px-3 py-1 rounded-md`, key decisions as a bullet list
   - Impact metrics: 2x2 grid, each metric value in `font-mono text-3xl font-bold text-[var(--color-accent)]`, label in `text-sm` secondary
   - Tags: row of `font-mono text-sm` pills
   - Links: Case study link and GitHub link as subtle text links with arrow, not big buttons

5. **Remove entirely:**
   - Tab button selector row (`flex-wrap justify-center gap-4`)
   - The big `apple-glass rounded-3xl` detail card wrapper (replaced by per-project expansion)
   - The `bg-gradient-to-br from-primary-blue/10 to-accent-purple/10` metric cards

**Skills.tsx -- Full-width list layout:**

Replace the 2-column `grid md:grid-cols-2 gap-8` layout with a single-column full-width stack. Each skill category becomes a full-width section rather than a card.

- Section header: Left-aligned (not centered). Title `text-4xl md:text-5xl font-heading font-bold`, accent word `text-[var(--color-accent)]`, subtitle below in `text-lg`.
- Each `SkillCategory` becomes a full-width block:

  - Skills inside each category: Use a `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6` for the individual skills (3-column on desktop, 2 on tablet, 1 on mobile) -- this creates asymmetry because different categories have different numbers of skills.
  - Each `SkillItem`: Left-aligned content block (not centered). Name in `font-semibold text-lg`, experience in `font-mono text-sm` with secondary color, proficiency dots + label on the same row, project context below in `text-sm italic`.
  - Remove `hover:scale-[1.02]` if any remains from Plan 02. Use `hover:border-[var(--color-accent)]` on individual skill items if they have a border.

**CareerTimeline.tsx -- Left-aligned timeline:**

Replace the centered horizontal desktop timeline with a left-aligned vertical layout on all screen sizes.

- Section header: Left-aligned.
- Desktop layout: Instead of 4 centered columns with horizontal line, use a **left-aligned vertical timeline**:
  - A thin vertical line on the LEFT side: `absolute left-8 top-0 bottom-0 w-px bg-[var(--color-border)]`
  - Each timeline entry is a row: year circle on the left (overlapping the line), content card to the right
  - Layout: `flex items-start gap-8` per entry, with the year badge `w-16 h-16 rounded-full flex-shrink-0` positioned on the left, and the content card flex-1 on the right
  - Content card: no card background on desktop (just text with bottom border `border-b border-[var(--color-border)] pb-8 mb-8`). Contains: highlight badge, role title, description bullets.
  - This creates an asymmetric left-heavy layout (line + circle on left, content flowing right)

- Mobile layout: Keep the existing mobile layout concept (left circle + right content), but simplify:
  - Remove the card wrapper (no bg, just border-bottom)
  - Year circle on left, content to right, vertical line connecting circles

- Remove the `activeIndex` state and click-to-expand behavior for desktop. On desktop, ALL entries are always visible (no selection needed -- the timeline tells a story top to bottom). On mobile, keep all entries visible too.
- Remove `useState` import if activeIndex is the only state.
- Keep `onClick` handlers removed -- timeline entries are not interactive on desktop.
- Year text in circles: already has `font-mono` from Plan 02.
- Highlight badge: keep the per-role `color` gradient on the small pill badge (these are the colored accents that give each role identity).
