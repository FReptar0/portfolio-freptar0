New @theme inline block:

```css
@theme inline {
  /* Typography */
  --font-heading: var(--font-heading);
  --font-body: var(--font-body);
  --font-sans: var(--font-body);
  --font-mono: var(--font-mono);

  /* Core Colors */
  --color-background: var(--color-bg);
  --color-foreground: var(--color-fg);

  /* Swiss Accent */
  --color-accent-green: var(--color-accent);
  --color-accent-green-hover: var(--color-accent-hover);
  --color-accent-green-muted: var(--color-accent-muted);

  /* Surfaces */
  --color-surface: var(--color-surface);
  --color-surface-elevated: var(--color-surface-elevated);
  --color-surface-sunken: var(--color-surface-sunken);

  /* Legacy color token aliases (components still use these Tailwind classes) */
  --color-primary-blue: var(--color-accent);
  --color-accent-purple: var(--color-accent);
  --color-success-green: var(--color-success);
  --color-growth-orange: var(--text-secondary);

  /* shadcn/ui compatibility tokens */
  --color-card: var(--card);
  --color-card-foreground: var(--card-foreground);
  --color-popover: var(--popover);
  --color-popover-foreground: var(--popover-foreground);
  --color-primary: var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  --color-secondary: var(--secondary);
  --color-secondary-foreground: var(--secondary-foreground);
  --color-muted: var(--muted);
  --color-muted-foreground: var(--muted-foreground);
  --color-accent: var(--accent);
  --color-accent-foreground: var(--accent-foreground);
  --color-destructive: var(--destructive);
  --color-border: var(--border);
  --color-input: var(--input);
  --color-ring: var(--ring);
  --color-chart-1: var(--chart-1);
  --color-chart-2: var(--chart-2);
  --color-chart-3: var(--chart-3);
  --color-chart-4: var(--chart-4);
  --color-chart-5: var(--chart-5);

  /* Sidebar tokens */
  --color-sidebar: var(--sidebar);
  --color-sidebar-foreground: var(--sidebar-foreground);
  --color-sidebar-primary: var(--sidebar-primary);
  --color-sidebar-primary-foreground: var(--sidebar-primary-foreground);
  --color-sidebar-accent: var(--sidebar-accent);
  --color-sidebar-accent-foreground: var(--sidebar-accent-foreground);
  --color-sidebar-border: var(--sidebar-border);
  --color-sidebar-ring: var(--sidebar-ring);

  /* Border radius */
  --radius-sm: calc(var(--radius) - 4px);
  --radius-md: calc(var(--radius) - 2px);
  --radius-lg: var(--radius);
  --radius-xl: calc(var(--radius) + 4px);
}
```

Key changes from the old @theme inline:
1. Font tokens now reference --font-heading, --font-body, --font-mono (NOT --font-geist-sans, --font-geist-mono)
2. --font-sans maps to --font-body (DM Sans) so any `font-sans` Tailwind usage gets the new body font
3. New Swiss surface tokens added (surface, surface-elevated, surface-sunken)
4. New accent green tokens added (accent-green, accent-green-hover, accent-green-muted)
5. Legacy color aliases preserved (primary-blue, accent-purple, success-green, growth-orange) mapped to new values
6. All shadcn/ui compatibility tokens preserved
7. Radius tokens preserved

IMPORTANT: There is a naming collision risk with `--color-accent` in @theme inline vs `--color-accent` in :root. The @theme inline `--color-accent` maps to the shadcn `--accent` token (which is a neutral color for hover states). The Swiss accent green is exposed as `--color-accent-green` in Tailwind. The raw CSS var `var(--color-accent)` in inline styles will still correctly resolve to #22C55E from :root.

After updating the @theme inline block, also verify that no other part of globals.css still references `--font-geist-sans` or `--font-geist-mono`. Remove any remaining stale references.
